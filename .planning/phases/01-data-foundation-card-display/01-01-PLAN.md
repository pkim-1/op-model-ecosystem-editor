Add the following JavaScript functions to the `<script>` block in index.html, after the CONFIG section. These implement the complete data layer.

**Token management functions (per D-02, D-03):**

`getToken()` -- returns `localStorage.getItem('github_token')` or `null`.

`saveToken(token)` -- calls `localStorage.setItem('github_token', token)`.

`clearToken()` -- calls `localStorage.removeItem('github_token')`.

**Header builder (per D-04):**

`getHeaders()` -- returns an object:
```javascript
function getHeaders() {
  const headers = { 'Accept': 'application/vnd.github+json' };
  const token = getToken();
  if (token) {
    headers['Authorization'] = `Bearer ${token}`;
  }
  return headers;
}
```

**Pagination-aware fetch (per D-04, D-05 -- handles >100 repos):**

`fetchAllRepos()` -- async function that:
1. Initializes `let allRepos = []; let page = 1;`
2. Loops: fetches `${CONFIG.apiBase}/users/${CONFIG.username}/repos?per_page=${CONFIG.perPage}&page=${page}&sort=updated&type=owner` with `getHeaders()`
3. Checks response status:
   - If 403 and response headers contain `x-ratelimit-remaining` equal to `"0"`: throw an Error with message `RATE_LIMITED` and attach the `x-ratelimit-reset` header value as `resetTime` property on the error
   - If 401: throw an Error with message `AUTH_FAILED`
   - If not ok: throw an Error with message `API_ERROR: ${response.status}`
4. Parses JSON: `const repos = await response.json();`
5. Pushes repos into allRepos: `allRepos.push(...repos);`
6. Break condition: `if (repos.length < CONFIG.perPage) break;`
7. Increments page
8. Returns `allRepos`

**Error display functions (per D-14, D-15, D-16):**

`showError(message, showRetry, showTokenInput)` -- function that:
1. Hides `#status` (`style.display = 'none'`)
2. Shows `#error` div with innerHTML containing:
   - A `<p>` with the message text (use textContent for the message to prevent XSS, then set it)
   - If `showRetry`: a `<button onclick="initDashboard()">Retry</button>`
   - If `showTokenInput`: a `<button onclick="showTokenPrompt()">Add GitHub Token</button>`
   Actually, build this with DOM methods (createElement) to avoid XSS from error messages. Create a div, create a p element, set p.textContent = message. Append p to div. If showRetry, create button with textContent "Retry", add click listener calling `initDashboard()`. If showTokenInput, create button with textContent "Add GitHub Token", add click listener calling `showTokenPrompt()`. Set `error.innerHTML = ''` then append the div.

`showTokenPrompt()` -- function that:
1. Hides `#error`
2. Shows `#token-prompt` with this content (built via DOM methods):
   - `<p>` with textContent "Enter a GitHub Personal Access Token for higher rate limits (5,000 requests/hour)."
   - `<p>` with textContent "Create one at github.com/settings/tokens with public_repo scope."
   - An `<input type="password" id="token-input" placeholder="ghp_xxxxxxxxxxxx">`
   - A `<button>` "Save Token" that on click: reads input value, calls `saveToken(value)`, hides token-prompt, calls `initDashboard()`
   - A `<span>` "Skip (continue without token)" that on click: hides token-prompt, calls `initDashboard()`

**Main initialization (per D-11, D-12, D-13):**

`initDashboard()` -- async function that:
1. Shows `#status` with textContent "Loading repositories..." (per D-11)
2. Hides `#error`, `#token-prompt`, clears `#project-grid`
3. Try block:
   - `const repos = await fetchAllRepos();`
   - Stores repos globally: `window.allRepos = repos;`
   - Hides `#status`
   - Updates `#repo-count` textContent to `Showing ${repos.length} repositories` (per D-13)
   - Calls `renderGrid(repos)` (this function will be implemented in Plan 02; for now, add a stub: `function renderGrid(repos) { console.log('Fetched', repos.length, 'repos', repos); }`)
4. Catch block handles errors by message:
   - `RATE_LIMITED`: call `showError('GitHub API rate limit reached. Unauthenticated requests are limited to 60 per hour.', true, true)`
   - `AUTH_FAILED`: call `clearToken()`, then `showError('GitHub token is invalid or expired. Please re-enter your token.', true, true)`
   - `API_ERROR: 404`: call `showError('GitHub user not found. Check the username in the dashboard configuration.', false, false)`
   - Default (network errors per D-16): call `showError('Failed to fetch repositories. Check your internet connection and try again.', true, false)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> GitHub API | Fetch requests cross to external API; responses are untrusted |
| User -> localStorage | PAT entered by user stored in browser storage |
| GitHub API -> DOM | Repo descriptions/names from API rendered to page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Token handling | mitigate | Store token in localStorage only (never in source). Use Authorization header (never query param). No hardcoded tokens. |
| T-01-02 | Spoofing | GitHub API response | accept | Low risk -- personal tool fetching own repos. API responses over HTTPS. |
| T-01-03 | Tampering | DOM injection via repo description | mitigate | Use textContent or createElement for all user-generated strings from API. Never inject raw API data via innerHTML. |
| T-01-04 | Denial of Service | Rate limiting | mitigate | Handle 403 + x-ratelimit-remaining=0 gracefully. Show token prompt to upgrade to 5000 req/hr. |
</threat_model>
