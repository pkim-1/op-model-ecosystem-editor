## Interaction Contract

### Badge Behavior

| Interaction | Behavior |
|-------------|----------|
| Click on badge | No separate action -- badge is inside card, card click opens repo (D-03) |
| Hover on card with badge | Standard card hover (translateY(-2px) + shadow) applies, badge moves with card |
| Badge render | Appears immediately with card -- no separate animation needed |
| No matching type | No badge rendered, no placeholder, no layout shift (D-06) |

### States

| State | Visual |
|-------|--------|
| Card with type match | Badge visible in top-left, title shifted down 24px |
| Card without type match | No badge, title at normal position (no layout change from current) |
| Card hover (with badge) | Badge lifts with card via transform, shadow unchanged |

## Type Detection Contract

### Keyword Map (D-04)

| Type | Keywords (case-insensitive substring match) |
|------|---------------------------------------------|
| Analysis | analysis, data-science, jupyter, research, statistics, analytics |
| Tool | tool, cli, library, framework, utility, automation, dashboard |
| Visualization | visualization, dashboard, chart, graph, plot, dataviz |

### Detection Rules

| Rule | Specification | Source |
|------|---------------|--------|
| Scan order | Topics array left to right (first match wins) | D-05 |
| Type priority | Analysis > Tool > Visualization (inner loop order) | D-04 order, RESEARCH.md Open Question 1 |
| No match | Return null, render no badge | D-06 |
| Case handling | Case-insensitive (toLowerCase before matching) | D-07 |
| Match method | Substring includes() on each topic | D-04 keywords contain compound strings |
