Create `tools/meeting-tracker.html` as a single self-contained HTML file with all CSS and JS inline. Structure:

**Data Model (localStorage):**
- `meetings` array: each meeting has `{ id, title, date, participants: [], items: [], rawNotes, createdAt }`
- Each item: `{ id, type: 'action'|'decision'|'discussion', text, assignee, project, status: 'open'|'done'|'pending', dueDate, createdAt }`

**Layout (CSS Grid, responsive):**
- Top header bar: EG Blue (#000099) background, white text, app title "Meeting Tracker"
- Left sidebar (collapsible on mobile): navigation with views -- Meetings, Timeline, People, Projects, Search
- Main content area: renders the active view

**Views to implement:**

1. **Meetings List View (default):** Cards showing each meeting with date, title, participant count, item counts by type. Click to expand/edit. "New Meeting" button prominent.

2. **New/Edit Meeting Form:**
   - Title, date, participants (comma-separated, autocomplete from history)
   - Two input modes with a toggle:
     a. **Manual entry:** Rich text area for notes, plus an "Add Item" form with type dropdown (action/decision/discussion), text, assignee dropdown (from participants), project tag, optional due date
     b. **Transcript paste:** Large textarea to paste Zoom transcript text. Parse button that extracts speaker names as participants, preserves full text as rawNotes, and uses simple heuristics to suggest categorization:
        - Lines with "action item", "TODO", "follow up", "will do", "needs to" => action item
        - Lines with "decided", "agreed", "decision", "we'll go with" => decision
        - Everything else => discussion
        - User can review and recategorize before saving

3. **Timeline View:** Vertical timeline sorted by date (newest first). Each node shows meeting title, date, and expandable item list. Color-coded dots: green for action items, purple for decisions, amber for discussions.

4. **People View:** List of all unique assignees/participants. Click a person to see all their action items (open and done), meetings they attended, grouped by meeting date.

5. **Projects View:** List of all unique project tags. Click to see all items tagged with that project across meetings.

6. **Search/Filter View:** Search bar with filters -- type (action/decision/discussion), status (open/done/pending), person, project, date range. Results show matching items with meeting context.

**Export functionality:**
- "Export" button on each meeting: generates a clean markdown summary copied to clipboard
- Format: `## Meeting: {title} ({date})\n### Participants\n{list}\n### Action Items\n{list with assignee}\n### Decisions\n{list}\n### Discussion Notes\n{list}`
- Also a "Export All Open Actions" button that exports all open action items across all meetings

**Persistence:**
- Save to localStorage on every mutation
- Load from localStorage on page init
- Import/export full data as JSON file (download button in sidebar footer)

**Styling:**
- CSS custom properties for all colors from EGA palette
- Card-based UI with subtle shadows and hover states
- Category badges: green pill for action items, purple for decisions, amber for discussions
- Status indicators: checkmark for done, circle for open, clock for pending
- Smooth transitions on view changes
- Responsive: sidebar collapses to hamburger menu on mobile (<768px)
- No external dependencies -- all inline CSS and JS
