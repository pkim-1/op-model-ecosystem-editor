<task type="auto">
  <name>Task 2: Add AI Analysis button and Claude API transcript analysis</name>
  <files>tools/meeting-tracker.html</files>
  <action>
In the transcript mode section of `renderMeetingForm`, add an "AI Analysis" button next to the existing "Parse Transcript" button. The AI Analysis button should be visually distinct (use `btn-primary` with a different label like "AI Analysis" while Parse Transcript stays `btn-secondary` or similar). If no API key is configured, the AI Analysis button should be disabled with a tooltip "Configure API key in Settings first".

Implement `async function aiAnalyzeTranscript()`:

1. Get transcript text from `#transcriptText`. If empty, show toast and return.
2. Read `claudeApiKey` from localStorage. If missing, show toast directing to Settings.
3. Show a loading state: disable the button, change text to "Analyzing..." and add a simple CSS spinner or pulsing dots.
4. Call the Claude API (`https://api.anthropic.com/v1/messages`) with:
   - model: `claude-sonnet-4-20250514`
   - max_tokens: 4096
   - system prompt instructing Claude to analyze the meeting transcript and return JSON with this exact structure:
     ```json
     {
       "participants": ["Name1", "Name2"],
       "topics": [
         {
           "name": "Topic Name",
           "items": [
             {
               "type": "action|decision|discussion",
               "text": "Description",
               "assignee": "Person or empty string",
               "status": "open|in-progress|blocked|waiting",
               "blocker": "Description of blocker or empty string",
               "project": "Auto-detected project/tool name or empty string",
               "context": "Brief context for decisions"
             }
           ]
         }
       ]
     }
     ```
   - The system prompt must instruct Claude to:
     - Extract action items with explicit owners
     - Detect implicit assignments: "I'll handle X" assigns to the speaker, "Erica is working on Y" assigns to Erica
     - Identify blockers and dependencies
     - Extract decisions with rationale/context
     - Group related items by topic/subject
     - Auto-detect project names, tool names, and systems mentioned
     - Set status: "open" for new items, "in-progress" for ongoing work, "blocked" if a blocker is mentioned, "waiting" if waiting on someone/something
   - Include the participants list (from the participants input field) in the user message so Claude knows who the speakers are

5. Parse the JSON response from Claude's message content. Handle cases where Claude wraps JSON in markdown code fences (strip them).

6. Error handling:
   - Network errors: show toast "Failed to connect to Claude API"
   - 401: show toast "Invalid API key - check Settings"
   - 429: show toast "Rate limited - try again in a moment"
   - JSON parse errors: show toast "Failed to parse AI response"
   - Wrap entire call in try/catch

7. On success, populate `_parsedItems` from the flattened topics structure (each topic's items become entries in `_parsedItems`), preserving the topic name as a prefix in the text or as a separate display element. Map the AI response fields to the existing item structure:
   - `type` maps directly
   - `text` maps directly (include `context` for decisions by appending it)
   - `assignee` maps directly
   - `status`: map "in-progress" to "pending", "blocked" to "pending", "waiting" to "pending" (existing app only has open/pending/done)
   - `project` maps directly
   - `blocker`: append to text in parentheses if non-empty, e.g., "Deploy new config (Blocked: waiting on DevOps approval)"

8. Call `renderParsedReview()` to show the review UI (reuse existing review flow so user can recategorize/remove before accepting).

9. Restore button state after completion (success or error).

CSS additions:
- `.btn-ai` style: background gradient or purple accent to distinguish from Parse Transcript
- `.loading-spinner` or simple animated dots for loading state
