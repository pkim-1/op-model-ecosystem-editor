<objective>
Fix the hub (yellow circle) drag behavior to prevent it from jumping to a different location when the user first grabs it. The hub should stay exactly where it is when mousedown occurs and then follow the cursor smoothly.
</objective>

<context>
The current implementation calculates the drag offset using the hub's bounding rect, but the hub is positioned using state.centerHub.x/y coordinates with a CSS transform: translate(-50%, -50%). This mismatch causes the hub to jump when dragging starts.

The offset calculation needs to be consistent with how the hub position is updated during mousemove (which uses state.centerHub coordinates accounting for zoom level).
</context>

<tasks>

<task type="auto">
  <name>Fix hub drag offset calculation</name>
  <files>Desktop/op-model-ecosystem-editor-LATEST.html</files>
  <action>
In the hub mousedown event handler (around line 1315), change the offset calculation to use the same coordinate system as the mousemove handler:

Replace:
```javascript
const rect = hub.getBoundingClientRect();
dragging = { type: 'hub', el: hub };
dragOffset.x = e.clientX - rect.left - rect.width / 2;
dragOffset.y = e.clientY - rect.top - rect.height / 2;
```

With:
```javascript
const containerRect = container.getBoundingClientRect();
dragging = { type: 'hub', el: hub };
dragOffset.x = (e.clientX - containerRect.left) / zoomLevel - state.centerHub.x;
dragOffset.y = (e.clientY - containerRect.top) / zoomLevel - state.centerHub.y;
```

This makes the offset calculation consistent with the mousemove handler (lines 1504-1509) which uses the same coordinate transformation.
