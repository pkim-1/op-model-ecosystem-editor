In the mousemove handler for hub dragging (around lines 1521-1522), change:
```javascript
const x = (e.clientX - containerRect.left) / zoomLevel - dragOffset.x / zoomLevel;
const y = (e.clientY - containerRect.top) / zoomLevel - dragOffset.y / zoomLevel;
```
To:
```javascript
const x = (e.clientX - containerRect.left) / zoomLevel - dragOffset.x;
const y = (e.clientY - containerRect.top) / zoomLevel - dragOffset.y;
```

The dragOffset was already calculated in world-space coordinates in the mousedown handler (mouse position divided by zoomLevel minus hub position). Dividing it by zoomLevel again in mousemove produces incorrect positioning. This fix matches the pattern used for card dragging.
